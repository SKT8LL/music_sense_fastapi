# TOOLS_RULES (antigravity/LLM + TDD + CI 규칙)

## 0. 전제
- 이 프로젝트는 “TDD + CI”를 기본 운영 원칙으로 한다.
- 도구(antigravity/LLM)는 초안 생성기이며, 최종 책임은 개발자에게 있다.

## 1. TDD 강제 조항
- 새 기능/수정은 항상 tests/에 테스트를 먼저 추가한다.
- 테스트가 실패(RED)하는 것을 확인한 후 구현한다.
- 테스트 없는 기능 PR은 원칙적으로 반려한다.

## 2. CI 게이트
- PR 오픈 시 CI가 pytest를 자동 실행한다.
- CI 실패 시 병합 금지.
- 로컬 pytest 통과 후 PR을 올린다.

## 3. 도구 사용 허용 범위
- 허용:
  - 테스트 코드 초안 생성
  - 단일 함수/헬퍼 구현
  - 에러 케이스 정리, 문서/명세 초안
- 금지:
  - 프로젝트 전체 생성/구조 변경을 도구에 일임
  - requirements.txt 임의 변경
  - ORM/마이그레이션 자동 생성 그대로 적용
  - 보안/인증 복잡 기능 자동 도입

## 4. 패키지 정책
- 패키지 추가는 “요청 → 승인(김상엽) → 반영” 절차만 허용.

## 5. 프롬프트 작성 원칙
- 범위 최소화: “파일 전체”가 아니라 “테스트 1개 파일” 또는 “함수 1개”
- 입력/출력/에러 케이스/예시를 명확히
- 결과물은 반드시 사람이 검토/수정 후 커밋
